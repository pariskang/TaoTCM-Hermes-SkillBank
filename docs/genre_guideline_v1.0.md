# 中医古籍文体分类标注规范 v1.0

**TaoTCM-Hermes AutoBuild Protocol · L0.5 文体路由层**

适用对象：人类标注员（micro-gold 构建与终审）、LLM 自动标注器（作为 system prompt / few-shot 规范）。

---

## 第一部分 总则

### 1.1 标注目标

对 `data.xlsx` 中每一行（row）的古籍文本，完成两件事：

1. **文体判定**：将文本判定为八类文体之一，或判定为混合文体；
2. **混合文体切分**：若为混合文体，给出字符级切分边界（span），每个 span 各归一类。

文体判定的下游后果是路由：文体决定该段文本进入哪个标注模板、能生成哪类知识资产、证据等级上限是什么。**因此文体标注的判断标准是"知识承载结构"，不是"内容主题"。** 同样讲麻黄汤，条文是规则、方后是档案、医案是实例、歌诀是助记——四种文体，四种命运。

### 1.2 八类文体标签

| 标签 | 中文名 | genre 代码 |
|---|---|---|
| 1 | 经论条文体 | `canonical_clause` |
| 2 | 医论体 | `treatise` |
| 3 | 方书体 | `formula_entry` |
| 4 | 本草体 | `materia_medica` |
| 5 | 脉学体 | `pulse_text` |
| 6 | 医案体 | `case_record` |
| 7 | 注释体 | `commentary` |
| 8 | 医话/歌诀体 | `mnemonic_misc` |

辅助标签（不可单独使用，须与主标签共现或作为整行标记）：

- `mixed`：整行含两类以上文体，须给出 segmentation；
- `genre_uncertain`：两轮判定仍无法归类，进入人工仲裁队列；
- `non_medical`：序跋、目录、刊刻牌记、避讳说明等非医学内容，跳过下游标注。

### 1.3 标注基本原则

1. **结构优先于主题**：依据文本的语法结构与功能标志词判定，不依据其讨论的医学主题。
2. **最小充分切分**：混合文体切分到"一个 span 一种文体"即止，不做更细的句内切分。
3. **疑义保留**：两类文体难以裁决时，标注 `genre_uncertain` 并填写 `uncertainty_note`，禁止强行二选一。
4. **原文不动**：所有 span 以原文字符 offset 定位，标注过程不得改写、标点重排或简繁转换原文（归一化在 L1 进行）。
5. **来源即上下文**：允许并应当利用 `book / volume / chapter` 字段辅助判断（如来源为《脉经》则脉学体先验概率高），但来源不能单独决定文体——《伤寒论》中也有方书体和医论体段落。

## 第二部分 八类文体的操作定义与判别特征

每类文体给出：操作定义、正向判别特征（满足越多越可信）、负向特征（出现则降低该类可能性）、典型标志词、易混淆对。

### 2.1 经论条文体 `canonical_clause`

**操作定义**：以"病位/病名 + 症候列举 + 处置断言"为骨架的规则性短文。核心是存在一个**可执行的条件→处置映射**，处置可以是方剂、治法或禁律。

**正向特征**：
- F1 句首为六经病名、病名或"病人/伤寒/中风/太阳病"类主语；
- F2 中段为顿列式症候（症状、脉象并列，常以"者"收束条件）;
- F3 句末出现处置断言词：**主之 / 宜 / 可与 / 不可与 / 当 / 可发汗 / 不可下 / 与之则X**；
- F4 全文较短（多数 < 80 字），无第一人称议论。

**负向特征**：
- 出现"愚谓 / 盖 / 夫 / 何也 / 此所以"等议论引导词 → 倾向医论或注释体；
- 出现具体人名、年龄、诊次 → 倾向医案体。

**判定规则**：F3 是近乎充分的标志——含"主之/宜/不可与"等断言词且有条件列举者，判为条文体。仅有 F1+F2 而无处置断言的（如纯病机描述条文，"病有发热恶寒者，发于阳也"），仍判条文体，但在下游标注 `clause_subtype: theoretical`（此类条文生成病机声明而非方证规则）。

**易混淆对**：见 3.1（条文 vs 医论）、3.4（条文+方后的混合切分）。

### 2.2 医论体 `treatise`

**操作定义**：阐述病机、治则、医理的议论性文本。核心是**主张（claim）而非规则（rule）**——它告诉你"为什么、是什么"，不直接告诉你"见X用Y"。

**正向特征**：
- F1 议论引导词：夫、盖、凡、故、是以、所谓、何也、答曰、愚按、论曰；
- F2 因果/机制连接词密集：故、则、所以、由是、致;
- F3 泛指主语（"人之、邪之、阳气者"）而非具体病人；
- F4 成段铺陈，常含排比（"诸X皆属于Y"句式）。

**负向特征**：
- 出现处置断言词"主之/宜服" → 检查是否混合了条文体；
- 出现具体诊疗经过 → 倾向医案。

**特别提示**：《内经》"诸风掉眩，皆属于肝"类病机十九条，虽形似规则，**判为医论体**——其"皆属于"是病机归类主张，不是临床处置规则。这是本规范最重要的一条人为裁定，目的是防止下游把哲学性归纳编译为硬匹配规则。

### 2.3 方书体 `formula_entry`

**操作定义**：以单一方剂为单元的结构化档案文本，含组成、剂量、炮制、煎法、服法、药后将息、加减法中的一项或多项。

**正向特征**：
- F1 药物+剂量顿列："麻黄三两（去节）　桂枝二两（去皮）"；
- F2 煎服法套语：**右X味，以水X升，煮取X升，去滓，温服X合**；
- F3 将息法："覆取微似汗 / 啜热稀粥 / 禁生冷"；
- F4 加减法："若渴者，去X加Y"。

**负向特征**：出现症候条件+断言词 → 混合了条文体，须切分。

**判定规则**：F2 套语（"右X味，以水…"）为充分标志。F4 加减法注意：加减法虽含条件句式（"若渴者"），仍归方书体，在模板内作为 `modification_rules` 字段，不切出为条文体。

### 2.4 本草体 `materia_medica`

**操作定义**：以单味药物为单元的属性描述文本，含性味、归经、功效、主治、毒性、炮制、产地、形态等。

**正向特征**：
- F1 条目以药名起首，紧跟性味："麻黄，味苦温"；
- F2 主治功效顿列："主X、Y、Z"；
- F3 出现"有毒/无毒/微毒"声明；
- F4 形态产地描述（"生于山谷 / 二月八月采"）。

**负向特征**：多味药并列+剂量 → 方书体；以方剂为单元 → 方书体。

**与方书体的边界**：单元是"一味药"→本草；单元是"一首方"→方书。《本草纲目》附方部分按方书体切出。

### 2.5 脉学体 `pulse_text`

**操作定义**：以脉象为单元的体征定义、指感描述、相类鉴别、主病归纳文本。

**正向特征**：
- F1 条目以脉名起首+指感定义："浮脉，举之有余，按之不足"；
- F2 指法术语：举、按、寻、推、初持、久按；
- F3 部位术语：寸、关、尺、人迎、趺阳；
- F4 相类鉴别句式："浮与芤相类 / 何以别之"；
- F5 主病句式："浮为风，为虚"。

**负向特征**：脉象仅作为症候之一出现在条件列举中（"脉浮紧"嵌在条文里）→ 不是脉学体，是条文体的组成部分。

**判定规则**：关键是**脉象是否为文本的论述单元**。"脉浮者，病在表，可发汗"——脉是条件、有处置断言，判条文体；"浮脉，举之有余"——脉是被定义对象，判脉学体。

### 2.6 医案体 `case_record`

**操作定义**：记录具体患者具体诊次的叙事性文本，含患者信息、病史时间线、医家辨析、处方、转归中的多项。

**正向特征**：
- F1 患者标识：姓氏/某/年龄/性别/籍贯（"某，年四十 / 王姓妇 / 一人患…"）；
- F2 时间线词：初诊、复诊、又、翌日、三剂后、越数日；
- F3 叙事时态：先…继而…遂…；
- F4 转归描述：而愈、热退、霍然、不起、卒;
- F5 医家第一人称介入："余诊之 / 予曰"。

**负向特征**：无具体患者、泛指议论 → 医论或医话。

**与医话的边界**：见 3.2。核心判据：**有无可重建的"单个患者诊疗时间线"**。有 → 医案；只是借事说理、患者信息残缺到无法重建诊疗过程 → 医话。

### 2.7 注释体 `commentary`

**操作定义**：依附于某一目标原文（条文、经文）展开解释、发挥或驳议的文本。核心是**存在明确的被注对象**。

**正向特征**：
- F1 引文+注释结构：先引原文（常顶格或加"经曰/论曰"），后接注语；
- F2 注释引导词：注曰、按、愚谓、成注、程注、所以然者；
- F3 训诂句式："X者，Y也 / X，谓Y"；
- F4 驳议句式："旧注以为X，非也"。

**负向特征**：无被注对象的独立议论 → 医论体。

**判定规则**：必须能指认 `target_clause`（被注的是哪段原文）。指认不出被注对象的"注家议论"，判医论体并在 school_tag 记注家归属。注释体中**被引的原文 span 切出为对应原文文体**（通常是条文体），注语部分才是注释体——见 3.5。

### 2.8 医话/歌诀体 `mnemonic_misc`

**操作定义**：两个亚类合并。**歌诀亚类**：为记忆服务的韵文（汤头、药性赋、脉诀韵语）；**医话亚类**：笔记式杂谈、见闻、读书心得、医林掌故。共同点是证据等级低（E5）、只进教学素材池、永不生成规则。

**歌诀正向特征**：句式整齐（五言/七言为主）、押韵、内容为方剂组成/药性/脉象的压缩复述。
**医话正向特征**：笔记体（"尝见 / 闻之某翁 / 偶读"）、夹叙夹议、单则短小、无系统结构。

**亚类字段**：`sub_genre: verse | note`。

**与脉学体的边界**：《濒湖脉学》体例歌诀（"浮脉惟从肉上行"）——韵文形式判 `mnemonic_misc/verse`，但其 `target` 字段链接到对应脉象本体；李时珍在歌诀后所附散文体定义则判脉学体。即：**同一书内按 span 文体各自归类，不按全书一刀切**。

## 第三部分 边界裁决规则（五对高频混淆）

### 3.1 条文体 vs 医论体

判据链（按序检查，命中即裁决）：
1. 有处置断言词（主之/宜/不可与/当刺）→ 条文体；
2. 无处置断言，但为"条件→病机/病名判定"的短规则（"…者，名为中风"）→ 条文体（`theoretical` 亚型）；
3. 议论引导词起首、泛论机理、成段铺陈 → 医论体；
4. 仍难裁决 → `genre_uncertain`。

### 3.2 医案体 vs 医话体

判据：能否从文本重建【患者 → 症候 → 干预 → 转归】四元组中至少三元？能 → 医案；不能（仅借一事议一理）→ 医话。
示例倾向：叶氏《临证指南》逐案 → 医案；《冷庐医话》忆某翁一语 → 医话，即便其中提及某个病人。

### 3.3 注释体 vs 医论体

判据：能否指认具体被注原文（target_clause 可定位）？能 → 注释体；不能 → 医论体。
注意：注家专著（如《注解伤寒论》）默认逐条对应，target 可由章节结构推断；医家论著（如《脾胃论》）即便频繁引经，引文是论据而非被注对象，整体仍判医论体，引文 span 单独切出。

### 3.4 条文体 + 方书体的切分（最高频混合）

《伤寒论》典型结构："…麻黄汤主之。【切分点】麻黄三两（去节）…右四味，以水九升…"。
切分规则：处置断言词（主之/宜X汤）属条文体 span 末尾；药物剂量列表起首即方书体 span 开始。两 span 间建立 `cross_link: {relation: "prescribes"}`。

### 3.5 注释体中的引文切分

结构："【原文引文 span → canonical_clause】【注语 span → commentary】"。
引文 span 标 `quoted: true` 且 `dedup_target` 指向语料库中该条文的主 fragment_id（避免同一条文因被多家引用而重复生成模板）。引文与底本有文字出入时，标 `variant_note`，这是异文研究的副产品。

### 3.6 通用兜底

- 单 span 文字 < 6 字且无法独立判定 → 并入相邻 span；
- 整行皆难判 → `genre_uncertain` + 必填 `uncertainty_note`（写明在哪两类之间摇摆、缺什么信息）；
- `genre_uncertain` 率应 < 5%；若某书超过 10%，说明该书文体超出八类体系，须上报扩展类目（如官修医书的诏令公文体、医经的问答体可能需要增设）。

## 第四部分 混合文体切分规范

1. **切分单位**：字符级 offset（含标点），span 之间不重叠、可不连续覆盖（非医学杂质可留空隙并标 `non_medical`）。
2. **最小 span**：原则上一个完整句读单元（以。；为界）；禁止句内切分，除非是 3.4 类断言词与药物列表的紧邻边界。
3. **cross_links 必填场景**：条文↔方剂（prescribes）、注释↔被注条文（comments_on）、歌诀↔方剂/脉象（mnemonic_of）、医案诊次↔所用方剂（applies）。
4. **顺序保留**：segmentation 数组按原文顺序排列，span 序号即阅读序。

## 第五部分 标注流程与质量控制

### 5.1 micro-gold 构建流程

1. 抽样：每类文体 10–20 段 + 混合文体 20 段，共 100–150 段，覆盖 MVP 涉及书目；
2. 双人独立标注（文体标签 + 切分边界）；
3. 一致性计算：文体标签报 Cohen's kappa（目标 ≥ 0.80，文体分类特征明显，应可达到）；切分边界报字符级边界 F1（目标 ≥ 0.85）；
4. 分歧仲裁：第三人裁决，裁决理由写入本规范"判例库"附录，规范随判例增长而版本化。

### 5.2 LLM 自动标注器校准

- 本规范第二、三部分直接作为标注器 system prompt 的判定准则部分;
- micro-gold 中每类抽 3–5 例作 few-shot；
- 双模型交叉标注，标签分歧或边界偏移 > 10 字符的样本进入 `model_disagreements` 队列，由终审包呈现；
- 标注器须输出 `confidence ∈ {high, medium, low}`；low 一律进审核队列。

### 5.3 输出合规

所有输出必须通过 `genre_segmentation.schema.json` 校验；schema 校验失败视为构建错误而非标注分歧，直接退回重跑。
